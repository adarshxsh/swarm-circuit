1. Executive Summary & Product Vision

SwarmCircuit v2 is an autonomous multi-agent game development platform where specialized AI workers collaborate to research, design, implement, review, and continuously improve game projects. Built specifically for independent studios and ambitious game engineers, it transforms raw community feedback into tested, production-ready code patches.

Godot Engine Focus

Full GDScript AST support, `.tscn` scene hierarchy parsing, and node connection validation.

Reddit Intelligence

Scans target subreddits for bug reports and meta-trends to generate prioritized engineering work.

Stateless Workers

Zero hallucination loops. Workers execute discrete tasks via strict JSON schema artifacts.

2. System Architecture & Workflow

SwarmCircuit operates on a strict Directed Acyclic Graph (DAG) orchestration model. Tasks flow from intent analysis into parallel execution tiers, ending at an executive synthesis for human review.

[ Godot Workspace ] ──> [ Project Chat ] ──> [ Intent Analysis ] ──> [ Deterministic Planner ]
                                                                                │
[ Reddit Threads ] ──> [ Reddit Scout ] ──> [ Community Analyst ] ──────────────┘
                                                                                ▼
[ Executive Reviewer ] <── [ Parallel Workers (GDScript, QA, Perf) ] <── [ DAG Scheduler ]
          │
          ▼
[ Creative Director Approval ] ──> [ Continuous Memory (.swarm) ]

4. Antigravity Pro Model Routing Matrix

SwarmCircuit dynamically assigns AI tiers based on reasoning requirements, latency constraints, and context lengths.

High-Reasoning Tier Pro High

Assigned to Planner, Architect, and Executive Reviewer for complex graph resolving and system abstraction.

Specialized Coding Coding Tier

Assigned to Gameplay Engineer and Perf Reviewer for AST generation and precise git diffing.

Cerebras Ultra-Fast 1000+ tok/s

Assigned to Continuous Self-Healing Engine for background scanning of dead code and project drift.

5. Continuous Project Memory Schema

To prevent hallucination across stateless runs, persistent context is structured into versioned files within .swarm/memory/.

.swarm/memory/
├── project_bible.json       # Core vision, rules, and global architecture invariants
├── architecture_graph.json  # Dependency tree and module boundaries
├── decision_log.jsonl       # Immutable ledger of accepted/rejected decisions
├── technical_debt.json      # Known shortcuts, FIXME tracking, and refactoring backlog
└── known_risks.json         # Security, performance, and scaling bottlenecks

6. Hybrid Execution Observatory

The studio dashboard supports dual-mode streaming to facilitate fast demos and live cloud inference:

• Live Mode: Connects via Server-Sent Events (/stream?mode=live) to execute real LLM tasks and update files.
• Demo Mode: Reads cached execution trajectories (golden_run.json) for instant zero-latency client-side playback.

7. 12-Month Development Roadmap (Phases 1–5)

The roadmap outlines the technical progression from foundational orchestration to enterprise-scale distributed cloud AI studios.

Phase 1: Foundation M 1–3

Deterministic DAG Orchestration, MVP Studio, Tree-sitter parsing, and exponential backoff retry engines.

Phase 2: Memory Layer M 4–5

Continuous Project Memory (.swarm schema), immutable decision logging, and AST slicing algorithms.

Phase 3: Specialist Fleet M 6–8

Deploying all 9 AI workers, Reddit Scout ingestion pipelines, and GitHub/Reddit MCP plugins.

Phase 4: Autonomous Engine M 9–10

Cerebras self-healing background scanning loops (1,000+ tok/sec), dead code pruning, and automated PR generation.

Phase 5: Studio Dashboard UI & Enterprise Scalability M 11–12

• Interactive Studio UI: Visual Web Studio replacing CLI output with websocket timelines, active worker graph inspection, and one-click PR reviews.
• Cloud Scaling & Storage: S3 blob storage for game artifacts, multi-user role-based access control (RBAC), and team collaboration workspaces.
• Distributed Workers: Celery / Redis distributed task orchestration executing parallel multi-agent loops across concurrent repositories.
